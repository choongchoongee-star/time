# 시간 가계부 (Time Ledger) — 기획·설계 명세서

> 이 문서는 **이 파일 하나만으로 동일한 앱을 처음부터 재구현**할 수 있도록 작성된 단일 기준 문서다.
> 기능, 데이터 구조, 화면, 상호작용, 계산 규칙, 예외 처리, 배포까지 모두 포함한다.

---

## 1. 개요

### 1.1 목적
하루 24시간을 "어디에 썼는지" 기록·확인하는 **개인용 시간 가계부**. 돈 가계부가 지출을 카테고리별로 남기듯, 이 앱은 **활동에 쓴 시간**을 카테고리별로 남기고 비율을 보여준다.

### 1.2 사용자 / 사용 맥락
- 사용자: **1인 (본인)**. 다중 사용자·공유·협업 없음.
- 기기: 주로 **iPhone Safari**. "홈 화면에 추가"로 앱처럼 사용.
- 네트워크: 최초 로딩 후 **오프라인에서도 동작**해야 함.

### 1.3 설계 원칙
1. **단순함 최우선.** 기본 기능만. 복잡한 설정/통계는 넣지 않는다.
2. **서버·로그인 없음.** 데이터는 기기 브라우저(localStorage)에만 저장.
3. **단일 파일 우선.** 앱 본체는 `index.html` 하나(HTML+CSS+JS 인라인). 의존 라이브러리 없음(바닐라 JS).
4. **모바일 퍼스트.** 세로 화면, 한 손 조작, 큰 터치 타깃.

### 1.4 비목표 (v1에서 만들지 않는 것)
- 계정/동기화/클라우드 저장
- 실시간 타이머(시작·정지 버튼으로 자동 측정) — *직접 입력 방식만 채택*
- 알림/리마인더, 위젯
- 월간/연간 통계, 그래프 차트(막대 비율 외), 데이터 내보내기를 CSV 등으로
- 다국어 (한국어 전용)

## 2. 핵심 개념과 데이터 모델

### 2.1 엔터티

#### Category (카테고리)
| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 ID. 기본값은 `c_sleep` 등, 사용자 추가분은 `c_<숫자>_<index>` |
| `name` | string | 표시 이름 (최대 12자) |
| `color` | string | HEX 색상 (`#rrggbb`). 점·막대 색으로 사용 |

#### Entry (기록)
| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 ID (`e_<timestamp>_<n>_<rand>`) |
| `date` | string | 기록 날짜 `YYYY-MM-DD` (로컬 기준) |
| `category` | string | Category.id 참조 |
| `start` | string | 시작 시각 `HH:MM` (24시간제) |
| `end` | string | 종료 시각 `HH:MM` |
| `minutes` | number | 소요 분(자동 계산, 저장 시 확정) |
| `memo` | string | 한 줄 메모 (선택, 빈 문자열 가능) |

| id | name | color |
|------|------|--------|
| c_sleep | 수면 | `#6366f1` |
| c_work | 일 | `#0ea5e9` |
| c_study | 공부 | `#22c55e` |
| c_exercise | 운동 | `#f97316` |
| c_rest | 휴식 | `#a855f7` |
| c_meal | 식사 | `#eab308` |
| c_etc | 기타 | `#94a3b8` |

### 2.3 localStorage 키
| 키 | 내용 |
|------|------|
| `tlb_entries_v1` | `Entry[]` (JSON) |
| `tlb_categories_v1` | `Category[]` (JSON) |

- 읽기 실패(파싱 오류/없음) 시 안전한 기본값으로 폴백.
- 카테고리가 비어 있으면 기본 카테고리로 재시드.

### 2.4 백업 파일 포맷 (내보내기/불러오기)
```json
{
  "version": 1,
  "exportedAt": "<ISO8601>",
  "categories": [ /* Category[] */ ],
  "entries": [ /* Entry[] */ ]
}
```
- 파일명: `시간가계부-백업-YYYY-MM-DD.json`
- 불러오기 시 `entries`와 `categories` 키가 모두 있어야 유효. 없으면 오류 메시지.

## 3. 계산 규칙

### 3.1 소요 시간 (minutes)
`start`, `end`를 분으로 변환(`HH*60+MM`) 후 `end - start`.
- 결과가 **0 이하이면 +1440(24시간) 보정** → 자정을 넘는 활동(예: 23:00→01:00 = 120분) 지원.
- 즉 `end == start`인 경우는 1440분(24시간)으로 간주.

### 3.2 시간 포맷 (fmtDur)
- 분 단위 입력 → `h시간 m분` 형태 표시.
  - h>0, m>0 → `"3시간 20분"`
  - h>0, m=0 → `"3시간"`
  - h=0 → `"20분"`

### 3.3 하루 합계 / 남은 시간
- `dayTotal` = 선택 날짜의 모든 Entry.minutes 합.
- `dayLeft` = `1440 - dayTotal`. 음수면 `"0분"` 표시(과기록 허용하되 남은시간은 0으로).

### 3.4 카테고리별 요약
- 범위 = `오늘`(선택 날짜) 또는 `이번 주`.
- **주 정의: 월요일 시작 ~ 일요일 끝** (선택 날짜가 포함된 주).
- 카테고리별 minutes 합산 → 내림차순 정렬 → 각 항목 비율 `round(min/총합*100)%`.
- 총합이 0이면 안내 문구 표시.

## 4. 화면 구성 & 와이어프레임

전체는 **단일 화면(스크롤)** + 두 개의 모달 시트(바텀시트)로 구성된다.

### 4.1 메인 화면

```
┌──────────────────────────────────────┐
│  ‹            오늘            ›        │  ← 헤더(sticky). 좌우 화살표로 날짜 이동
│            6월 14일 (토)              │     가운데 날짜 탭 → 오늘로 이동
│   기록 시간 1시간 30분 · 남은 22시간 30분 │
├──────────────────────────────────────┤
│  ┌────────────────────────────────┐  │
│  │        +  기록 추가             │  │  ← 항상 보이는 인라인 추가 버튼(본문 최상단)
│  └────────────────────────────────┘  │
│                                        │
│  오늘의 기록                           │  ← 섹션 타이틀
│  ┌────────────────────────────────┐  │
│  │ ● 공부            1시간 30분     │  │  ← Entry 카드(탭하면 수정 시트)
│  │   09:00 – 10:30 · 알고리즘 공부  │  │     왼쪽 점은 카테고리 색
│  └────────────────────────────────┘  │
│  ┌────────────────────────────────┐  │
│  │ ● 운동              40분         │  │
│  │   18:00 – 18:40                  │  │
│  └────────────────────────────────┘  │
│                                        │
│  카테고리별 요약                       │
│  ┌──────────────┬──────────────┐      │
│  │   [오늘]      │   이번 주     │      │  ← 토글(선택된 쪽 강조)
│  └──────────────┴──────────────┘      │
│  ● 공부            1시간 30분 · 69%    │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░                  │  ← 비율 막대(카테고리 색)
│  ● 운동              40분 · 31%        │
│  ▓▓▓▓▓▓░░░░░░░░░░░░░░░                │
│                                        │
│      카테고리 관리  백업 내보내기  불러오기 │  ← 하단 텍스트 링크
│                                        │
│                                  ╭───╮ │
│                                  │ + │ │  ← 플로팅 버튼(FAB, 화면 우하단 고정)
│                                  ╰───╯ │     인라인 버튼과 동일 동작
└──────────────────────────────────────┘
```

빈 상태:
- 기록 없음 → `"기록이 없어요. + 버튼으로 추가해 보세요."`
- 요약 데이터 없음 → `"기록이 쌓이면 여기에 비율이 보여요."`

### 4.2 기록 추가/수정 시트 (바텀시트)

```
┌──────────────────────────────────────┐
│  기록 추가                  (또는 수정) │
│                                        │
│  카테고리                              │
│  (수면) (일) (공부✓) (운동) (휴식) ...  │  ← 칩 선택, 선택된 칩 테두리 강조
│                                        │
│  ┌─ 시작 ──────┐  ┌─ 종료 ──────┐     │
│  │   09:00     │  │   10:30     │     │  ← time input (네이티브 시간 피커)
│  └─────────────┘  └─────────────┘     │
│  소요 시간 1시간 30분                   │  ← 입력 즉시 미리보기 갱신
│                                        │
│  메모 (선택)                           │
│  ┌────────────────────────────────┐  │
│  │ 알고리즘 공부                    │  │
│  └────────────────────────────────┘  │
│                                        │
│  ┌─── 취소 ───┐  ┌─── 저장 ───┐       │
│  └────────────┘  └────────────┘       │
│  ┌──────── 이 기록 삭제 ────────┐      │  ← 수정 모드에서만 표시
│  └──────────────────────────────┘      │
└──────────────────────────────────────┘
```

- 추가 모드 기본값: 카테고리=첫 번째, 시작=현재시각-30분, 종료=현재시각, 메모=빈칸.
- 배경(오버레이) 탭 → 닫기(취소와 동일).

### 4.3 카테고리 관리 시트

```
┌──────────────────────────────────────┐
│  카테고리 관리                         │
│  ● 수면                        삭제    │
│  ● 일                          삭제    │
│  ● 공부                        삭제    │
│  ...                                   │
│  ┌──┐ ┌──────────────────┐ ┌──────┐  │
│  │🎨│ │ 새 카테고리 이름    │ │ 추가 │  │  ← 색상 picker + 이름 + 추가 버튼
│  └──┘ └──────────────────┘ └──────┘  │
│                                        │
│  ┌──────────── 닫기 ─────────────┐    │
│  └──────────────────────────────┘     │
└──────────────────────────────────────┘
```

- 삭제 시 확인. 마지막 1개는 삭제 불가(`"카테고리는 최소 1개는 있어야 해요."`).
- 카테고리 삭제해도 기존 Entry는 보존되며, 사라진 카테고리는 목록/요약에서 `"(삭제됨)"`, 회색(`#94a3b8`)으로 표시.

## 5. 상호작용 명세 (이벤트 → 동작)

| 트리거 | 동작 |
|--------|------|
| 헤더 `‹` | 선택 날짜 -1일, 화면 재렌더 |
| 헤더 `›` | 선택 날짜 +1일, 재렌더 |
| 헤더 날짜 라벨 탭 | 선택 날짜를 오늘로 |
| 인라인 `+ 기록 추가` / FAB `+` | 추가 시트 열기(신규) |
| Entry 카드 탭 | 수정 시트 열기(해당 id) |
| 시트의 카테고리 칩 탭 | 선택 상태 갱신 |
| 시작/종료 input 변경 | 소요 시간 미리보기 갱신 |
| 시트 `저장` | 유효성 검사 후 추가/수정 → 저장 → 시트 닫고 재렌더 |
| 시트 `취소` / 오버레이 탭 | 시트 닫기(저장 안 함) |
| 시트 `이 기록 삭제` | 확인 후 삭제 → 저장 → 닫고 재렌더 |
| 요약 토글 `오늘`/`이번 주` | 요약만 재렌더 |
| `카테고리 관리` | 관리 시트 열기 |
| 관리 `추가` | 이름 유효성 검사 후 카테고리 추가 |
| 관리 항목 `삭제` | 확인(+최소 1개 보장) 후 삭제 |
| `백업 내보내기` | JSON 파일 다운로드 |
| `불러오기` | 파일 선택 → 검증 → 확인 후 전체 덮어쓰기 |

### 5.1 유효성 검사 (저장 시)
1. 카테고리 미선택 → `"카테고리를 선택해 주세요."` 후 중단.
2. 시작/종료 미입력 → `"시작/종료 시각을 입력해 주세요."` 후 중단.
3. 통과 시 `minutes` 계산하여 저장.

## 7. 기술 / 파일 구성

| 파일 | 역할 |
|------|------|
| `index.html` | 앱 전체. HTML 구조 + `<style>` + `<script>`(IIFE, 바닐라 JS). 외부 의존성 없음 |
| `manifest.json` | PWA 매니페스트(이름, 색상, 아이콘, `display: standalone`) |
| `sw.js` | 서비스워커. **네트워크 우선 + 캐시 폴백**(온라인=최신, 오프라인=캐시) |
| `icon-180.png` / `icon-192.png` / `icon-512.png` | 앱 아이콘(인디고 배경 + 흰 시계) |
| `.claude/launch.json` | 로컬 정적 미리보기 설정(`npx serve`) |
| `README.md` | 사용·배포 안내 |
| `SPEC.md` | (이 문서) |

### 7.1 PWA 동작
- `<meta name="apple-mobile-web-app-capable" content="yes">` 등으로 iOS 전체화면.
- 서비스워커는 최초 방문 시 핵심 자원 캐시. 이후 `fetch`는 네트워크 우선, 실패 시 캐시(`index.html` 폴백).
- 캐시 버전(`CACHE` 상수, 현재 `tlb-v2`)을 올리면 `activate`에서 이전 캐시 삭제.

### 7.2 코드 구조 (index.html 내 JS)
- 즉시실행함수(IIFE), `"use strict"`.
- 영역별 함수 그룹: storage(load/save) → date helpers(ymd, weekRange, minutesBetween, fmtDur) → render(renderEntries/renderSummary) → entry sheet(openEntry/saveEntry/deleteEntry) → category manager → backup(export/import) → event bindings → 초기 `render()`.
- 사용자 입력 문자열은 `escapeHtml`로 이스케이프 후 DOM 삽입(XSS 방지).

## 9. 예외 / 엣지 케이스

| 상황 | 처리 |
|------|------|
| localStorage 파싱 실패 | 기본값 폴백, 앱 정상 구동 |
| 자정 넘는 활동(23:00→01:00) | +1440 보정으로 120분 계산 |
| 하루 합계 > 24시간(과기록) | 허용, 단 "남은 시간"은 0분 |
| 삭제된 카테고리를 참조하는 기록 | `"(삭제됨)"` 회색으로 표시, 데이터 보존 |
| 백업 파일 형식 오류 | `"불러오기 실패: ..."` 메시지, 기존 데이터 유지 |
| 마지막 카테고리 삭제 시도 | 차단 |
| 앱 업데이트 후에도 옛 화면 | SW가 네트워크 우선이라 온라인 시 자동 최신화 |

## 10. 향후 확장 아이디어 (v1 범위 밖, 참고용)
- 분 단위 소요 시간 **직접 입력** 옵션(시작/종료 대신).
- 월간 요약 / 날짜 범위 선택.
- 카테고리 이름·색 **수정**(현재는 추가/삭제만).
- 목표 시간 설정 및 달성률.
- 기록 복제, 자주 쓰는 활동 즐겨찾기.
- iCloud/드라이브 자동 백업 연동.
